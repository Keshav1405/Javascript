Q2. What is an event loop and call stack?

An event loop is something that pulls stuff out of the queue and places it onto the global execution context whenever the GEC becomes empty.

Callstack is a way for the Java engine to keep track of its place in code that calls multiple functions.When you execute a script, the JavaScript engine creates a global execution context and pushes it on top of the call stack.Whenever a function is called, the JavaScript engine creates a function execution context for the function, pushes it on top of the call stack, and starts executing the function.When the current function completes, the JavaScript engine pops it off the call stack and resumes the execution where it left off.The script will stop when the call stack is empty.

Q4. What is a Temporal Dead Zone?

If you are accessing any variable before its declaration and that variable is defined using let and const, then you will be getting reference error because the variable is in temporal dead zone.

Q5. What is hoisting?

Hoisting in JavaScript is a behavior in which a function or a variable can be used before declaration and this is happening due to the creation phase of the Execution Context. It gives us an advantage that no matter where functions and variables are declared, they are moved to the top of their scope regardless of whether their scope is global or local. It allows us to call functions before even writing them in our code.

Q6. What is the DOM?

DOM stands for document object model and it is used to access the root file i.e the HTML file of the webpage. Using DOM, you can add new HTML elements as well as manipulate the existing elements.

Q7. Difference between undefined vs not defined vs NaN.

Undefined is a datatype in JS which means a variable is declared but its value is not specified. It is also the default value of a variable declared using 'var' upon hoisting.

Not defined is shown on the console when a variable is used for something but it was never declared.

NAN stands for 'not a number' which is the value given to a number if it contains alphabets or any special character. For ex: "23e4%89"
